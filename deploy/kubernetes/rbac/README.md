# Running platformctl against Kubernetes: RBAC

The `kubernetes` runtime (`spec.runtime.type: kubernetes`, behind the
`KubernetesRuntime` feature gate) authenticates with whatever the ambient
kubeconfig grants. Two postures:

## Dev shortcut: cluster-admin

Fastest path for a local cluster (minikube, kind) you already trust
yourself with:

```sh
kubectl create clusterrolebinding my-cluster-admin \
  --clusterrole=cluster-admin \
  --user="$(kubectl config view --minify -o jsonpath='{.users[0].name}')"
```

Your existing kubeconfig context already works; no further setup needed.

## Minimal production role

`role.yaml` + `serviceaccount.yaml` + `binding.yaml` grant exactly the
verbs the adapter uses — nothing else. The list is generated by reading
every `clientset.*` call in `internal/adapters/runtime/kubernetes/*.go` and
is kept in sync with `preflight.go`'s `preflightChecks` (the same list
`platformctl validate`/`plan` checks against at connect time, docs/planning/08
B6) — a stale RBAC manifest and a stale preflight check would drift apart
visibly, since a missing verb here shows up as a named preflight failure.

Apply and mint a kubeconfig scoped to the ServiceAccount's token:

```sh
kubectl apply -f serviceaccount.yaml -f role.yaml -f binding.yaml

# A short-lived token (1h here); re-run to refresh, or use a longer
# --duration for CI runs that outlast this.
TOKEN=$(kubectl create token platformctl -n platformctl-system --duration=1h)
SERVER=$(kubectl config view --minify -o jsonpath='{.clusters[0].cluster.server}')
CA=$(kubectl config view --minify --raw -o jsonpath='{.clusters[0].cluster.certificate-authority-data}')

kubectl config set-cluster platformctl-cluster --server="$SERVER" \
  --certificate-authority=<(echo "$CA" | base64 -d) --embed-certs=true \
  --kubeconfig=platformctl.kubeconfig
kubectl config set-credentials platformctl --token="$TOKEN" \
  --kubeconfig=platformctl.kubeconfig
kubectl config set-context platformctl --cluster=platformctl-cluster \
  --user=platformctl --kubeconfig=platformctl.kubeconfig
kubectl config use-context platformctl --kubeconfig=platformctl.kubeconfig

KUBECONFIG=platformctl.kubeconfig platformctl validate ./manifests
```

CI's Kubernetes integration job (`.github/workflows/ci.yml`) runs the full
K8s-tagged integration suite under exactly this role, against a fresh `kind`
cluster, to prove it's actually sufficient — not just documented as such.

## What each verb is for

| Resource | Verbs | Used by |
|---|---|---|
| `namespaces` | get, create, delete, list | `EnsureNetwork`/`RemoveNetwork` (a Provider's `runtime.network` maps to a Namespace) |
| `nodes` | list | `EnsureReachable`'s `node-port` access mode (resolves a node address) |
| `deployments.apps` | get, create, update, delete, list | `EnsureContainer`/`Remove`/`Inspect`/`ListManaged` (a container maps to a Deployment) |
| `statefulsets.apps` | get, create, update, delete, list | StableIdentity replica sets and the shape-transition guard (docs/adr/004): every `EnsureContainer`/`Inspect`/`ListManaged`/`Remove` consults both workload shapes |
| `poddisruptionbudgets.policy` | get, create, update, delete | applied automatically whenever `Replicas > 1` (docs/adr/004) |
| `services` | get, create, update, delete, list | `EnsureContainer`'s Service (DNS/addressing) + `EnsureReachable`'s `node-port`/`load-balancer` modes |
| `persistentvolumeclaims` | get, create, update, delete, list | `EnsureVolume`/`RemoveVolume` |
| `secrets` | get, create, update, delete | `ContainerSpec.Files`/`ImagePullAuth`, and the `KubernetesSecretBackend` SecretStore adapter |
| `pods` | get, list, create, delete | Finding a Deployment's live pod (logs, exec, port-forward); create/delete for `ProbeReachable`'s ephemeral probe pod fallback (docs/planning/08 C10) |
| `pods/log` | get | `Logs` |
| `pods/exec` | create | `ReadFile`'s live-path fallback, `ProbeReachable`'s exec-dial fast path |
| `pods/portforward` | create | `EnsureReachable`'s `port-forward` access mode (the default) |
| `networkpolicies.networking.k8s.io` | get, create, update, delete | `EnsureNetwork`'s default-deny + allow-same-namespace pair (opt out with `networkPolicy: none`) |
| `selfsubjectaccessreviews.authorization.k8s.io` | create | The `validate`/`plan`-time preflight check itself |
