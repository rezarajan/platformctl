# cdc-to-lake

Postgres → Debezium → Redpanda → S3 sink → MinIO. A database's changes
stream through change-data-capture into a topic, then land as objects in
an object store — the spine of most data platforms.

```
Source(app-db) ──Binding(cdc)──▶ EventStream(app-events)
                                          │
                                  Binding(sink)
                                          ▼
                                Dataset(raw-lake)  →  MinIO
```

Generated by `platformctl init cdc-to-lake`. Every manifest here already
validates as written — no edits required to run `platformctl validate`.

## Before you apply

1. **Build the S3 sink Connect image** (stock Kafka Connect images ship no
   S3 sink plugin):

   ```sh
   docker build -t datascape-s3sink-connect:local ./s3sink-image
   ```

2. **Fill in `.env`** with real credentials (placeholders are pre-filled
   for local/dev use — change them before anything you care about depends
   on this platform):

   ```sh
   $EDITOR .env
   ```

   Any key you leave unset is reported by name when you `apply` — secret
   `Preflight` checks every declared `SecretReference` before touching any
   infrastructure, so a missing credential never causes a half-applied
   platform.

## Run it

```sh
platformctl validate .
platformctl apply    . --env-file .env --auto-approve
platformctl status   .
```

Then generate some change traffic:

```sh
platformctl inventory .   # host ports and credentials pointers, once applied

psql postgres://admin:<password>@localhost:<db-port>/appdb \
  -c "CREATE TABLE records (id serial PRIMARY KEY, payload text);
      INSERT INTO records (payload) VALUES ('hello'), ('world');"

# objects appear under raw-events/events/ within ~30s
mc alias set local http://localhost:<minio-port> <lake-user> <lake-pass>
mc ls --recursive local/raw-events/
```

Host ports are auto-assigned from each resource's name (deterministic,
collision-avoiding) — run `platformctl inventory .` after applying to see
which port landed where; nothing here needs editing to pick one.

Tear down:

```sh
platformctl destroy . --auto-approve
```

## Capturing another table

Only tables listed on the CDC Binding's `options.tables` are captured
(`binding-cdc.yaml`). Add a table name and re-apply to widen capture
without recreating any infrastructure — see
`examples/cdc-attendance/README.md` for the full walkthrough of this
pattern (out-of-band failure recovery, drift healing, and the caveats
around late-declared tables) if you need more detail than this quickstart.
