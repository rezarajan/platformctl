# stream-basics

A managed Redpanda broker with two topics (`events`, `logs`) — the
smallest useful platformctl platform, for prototyping producers/consumers
or trying the tool out before wiring up a full CDC pipeline.

Generated by `platformctl init stream-basics`. Every manifest here already
validates as written — no edits required.

## Run it

```sh
platformctl validate .
platformctl apply    . --auto-approve   # no secrets to fill in for this one
platformctl status   .
platformctl inventory .                  # the auto-assigned Kafka host port
```

Point any Kafka client at the reported bootstrap address. Tear down:

```sh
platformctl destroy . --auto-approve
```

## Adding a topic

Add another `EventStream` manifest referencing `providerRef: {name:
broker}` and re-apply — the broker is not restarted, the topic is created
alongside the existing ones.

## Growing this into a full pipeline

`stream-basics` is deliberately minimal. For a database → stream → object
store pipeline, see `platformctl init cdc-to-lake` (or `--list` for every
shipped blueprint).
